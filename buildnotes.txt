dblatex build on OS X appears to be not easily available - need to install via Fink - Fink not responding on 7/28/2013. Use --fop option instead. 

a2x -v -fpdf --fop syllabus.txt

Apparently there is a bug that is triggered when you try to set the --destination-dir parameter to a different directory. Writing to the source directory for now. 

Adding some processing options for the xmlproc conversion of the generated DocText document to PDF

a2x -v -fpdf --fop --xsltproc-opts="--stringparam paper.type USletter" --xsltproc-opts="--stringparam body.font.master 10" --xsltproc-opts="--stringparam body.font.family sans-serif" syllabus.txt

Now, generate the corresponding HTML5 document as a complement to the PDF file

asciidoc -a data-uri -a icons -a toc -a max-width=55em -b html5 syllabus.txt
